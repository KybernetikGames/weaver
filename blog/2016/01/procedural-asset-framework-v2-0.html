Procedural Asset Framework v2.0

Published
Tuesday, 12 January 2016
Category
Release
Author
Kailas

The Full Version is available for $25.

The Lite Version is Free and includes all the features of the full version, but limits you to a maximum of 15 procedural assets per project.

At its core, the Procedural Asset Framework gives you the ability to procedurally generate assets using code and reference them using generics. The system is very open in how it can be used, allowing you to integrate it into your workflow as you see fit, for example:

  • If you create a PlayerPrefab : Procedural.Prefab class, you can reference the generated prefab in code using Static<PlayerPrefab>.Instance. Or if you want to use it like a regular prefab, you can still drag it into a scene or into a field in a component, or load it using Resources.Load like any regular prefab.
  • Alternatively, if you don't actually want to generate the asset using code, but you do want to be able to reference it using generics, you can make a PlayerPrefab : Procedural.ManualAsset<GameObject> class and give it the path to your asset (inside a Resources folder).

Other Features:

  • Procedural Layers Script: a procedurally generated script which contains an int constant corresponding to each of the layers in your project (including custom layers). This means that instead of LayerMask.NameToLayer("MyCustomLayer"), you can use Layers.MyCustomLayer. Not only is this more efficient, but it also means that there's no risk of getting the name wrong, and if you rename any of the layers you can simply regenerate the script, then the compiler will warn you about anywhere you are still using the old name so you can clean up your code.
  • Custom Layer Masks: you can also specify custom layer masks to be added to the script using the Layer Mask Manager window. Instead of using (1 << Layers.Default) | (1 << Layers.Player) | (1 << Layers.Enemy) ..., you can just make a custom mask and select the layers you want it to include, then reference it using Layers.MyCustomMask.
  • [PreBuild] Attribute: an attribute which can be placed on (almost) any method to have Unity call it automatically whenever you build the project. The attribute can be placed on any static method to have it called once at the start of a build. It can also be placed on an instance method in a MonoBehaviour class, which will cause it to be called once on every instance of that class in every scene included in the build.
  • Procedural App Details: automated build numbering (and build date), as well as runtime access to the product name and company name you specify in the Player Settings.

Questions, feedback, and feature requests are most welcome.
